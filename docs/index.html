Ploinky Research Agents

copilot-agents is a Ploinky agent repository that integrates Open Interpreter, Agentic Data Scientist, OpenHands, MLJAR, and DeepAnalyze as tagged research backends. Users invoke them from Copilot or WebMeet chat with @backend mentions; execution is relayed through researchRelay to provider agents that run local bwrap sandboxes from the shared bwrap-runner image. The suite is not a default AssistOSExplorer dependency. It deploys only through an explicit research-agents bundle.

Source of Truth

The DS specifications under docs/specs/ are the source of truth for contracts and invariants. Code, configuration, and tests must follow them. Update specs in the same change as any behavior, interface, or runtime configuration change. Spec-code synchronization is a repository invariant, not an optional documentation task.

Quick Start

cd <workspace-root>
ploinky add repo copilot-agents <repo-url-or-local-path>
ploinky enable repo copilot-agents
ploinky enable agent copilot-agents/research-agents global
ploinky start explorer 8080

The bundle enables researchRelay and openInterpreterAgent. Backend availability is controlled by provider agents, not by direct backend chat agents, a separate bwrap-runner agent, or relay-owned command strings.

Agent Inventory

  • research-agents - explicit deployment bundle
  • researchRelay - Copilot launch-extension and tagged task relay
  • openInterpreterAgent - Open Interpreter provider agent
  • @open-interpreter - active Open Interpreter backend tag

Chat Tags

Explorer exposes the normal Open Copilot here action. The researchRelay launch extension adds research-tags=1, forward-envelope=1, and explicit tag-relay parameters to that Copilot WebChat launch. Messages like @open-interpreter summarize this file are intercepted by AchillesCLI's configured tag-relay mode, sent to research_task_submit, delegated to openInterpreterAgent, executed inside that provider's local bwrap sandbox, and returned to the same chat. Unknown @word mentions remain ordinary chat. Explorer launch URLs use workspace-relative directory parameters so absolute host workspace paths do not appear in copied browser URLs. WebMeet MCP chat uses the same relay and appends the result as an agent-kind chat message.
