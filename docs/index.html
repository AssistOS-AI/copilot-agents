Ploinky Research Agents

copilot-agents is a Ploinky agent repository that integrates Open Interpreter, Agentic Data Scientist, OpenHands, MLJAR, and DeepAnalyze as provider-backed runtimes. AchillesCLI Copilot selects providers semantically from prompt and context; execution is relayed through copilotProviderRelay to provider agents that run local bwrap sandboxes from the shared bwrap-runner image. The suite is not a default AssistOSExplorer dependency. It deploys only through an explicit research-agents bundle.

Source of Truth

The DS specifications under docs/specs/ are the source of truth for contracts and invariants. Code, configuration, and tests must follow them. Update specs in the same change as any behavior, interface, or runtime configuration change. Spec-code synchronization is a repository invariant, not an optional documentation task.

Quick Start

cd <workspace-root>
ploinky add repo copilot-agents <repo-url-or-local-path>
ploinky enable repo copilot-agents
ploinky enable agent copilot-agents/research-agents global
ploinky start explorer 8080

The bundle enables copilotProviderRelay, openInterpreterAgent, and webSearchAgent. Backend availability is controlled by provider agents, not by direct backend chat agents, a separate bwrap-runner agent, or relay-owned command strings. webSearchAgent owns its local headless browser runtime and does not depend on an external search gateway.

Agent Inventory

  • research-agents - explicit deployment bundle
  • copilotProviderRelay - secure provider task relay
  • openInterpreterAgent - Open Interpreter provider agent
  • webSearchAgent - cacheable web-search provider with a local headless browser runtime
  • browserUseAgent - interactive browser provider with protected viewer for ChatGPT/Gemini login and OAuth flows
  • launch-open-interpreter - deterministic AchillesCLI launcher for execution tasks
  • launch-web-search - deterministic AchillesCLI launcher for cacheable web search
  • launch-browser-use - deterministic AchillesCLI launcher for logged-in browser tasks

Semantic Copilot Routing

Explorer exposes the normal Open Copilot here action. Copilot WebChat may forward a sanitized envelope and invocation token to AchillesCLI. AchillesCLI's copilot-router decides whether to answer normally or call a deterministic launcher such as launch-open-interpreter. The launcher submits to copilot_provider_task_submit, delegated to openInterpreterAgent, executed inside that provider's local bwrap sandbox, and returned to the same chat. Text such as @open-interpreter summarize this file is ordinary chat text, not a dispatch token. Search prompts route through launch-web-search to webSearchAgent, which performs local browser search and returns citations. Logged-in web application tasks route through launch-browser-use to browserUseAgent, which controls a persistent Chromium session and exposes a protected viewer URL for ChatGPT and Gemini login flows.
