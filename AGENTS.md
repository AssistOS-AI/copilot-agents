# copilot-agents

Ploinky agent repository for explicitly deployed research and development copilots (Open Interpreter, OpenHands, Agent Laboratory, The AI Scientist).

## Mandatory Reading Order

1. The workspace root `AGENTS.md` and any nested `AGENTS.md` for stricter local rules.
2. `docs/index.html` for the local documentation entry point.
3. `docs/specs/matrix.md` and the specific DS file(s) covering your change.
4. `docs/specs/DS002-ploinky-runtime-invariants.md` and `docs/specs/DS011-security-observability.md` before changing routing, manifests, MCP tools, file access, logs, or telemetry.
5. The sibling Ploinky and AssistOSExplorer specs referenced by the affected DS file when integration boundaries are involved.

## Repository Rules

- The DS specifications under `docs/specs/` are the source of truth for local contracts and invariants. Update them whenever code changes behavior, interfaces, or runtime configuration.
- Spec-code synchronization is an invariant: if code, tests, docs, manifests, or specs disagree, fix the implementation to match the affected DS file or update the DS file first with a numbered decision entry.
- `AGENTS.md` and `CLAUDE.md` at this level must stay identical.
- Keep DS numbering contiguous within this repository.
- Research agents must not be added to AssistOSExplorer's default dependency list. They deploy only through the `research-agents` bundle.
- Heavy execution agents must stay behind explicit bundle profiles.
- All technical documentation, specifications, and code comments must be in English.
- Never add Co-Authored-By, Generated by, or any other AI/coding-agent attribution to commits, documentation, code comments, release notes, changelogs, PR text, or other project records.

## Validation

Run the narrowest relevant check after edits:

- `node --test tests/unit/*.test.mjs`
- `node scripts/validate-manifests.mjs`
