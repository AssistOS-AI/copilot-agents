Documentation entry point for the copilot-agents Ploinky repository.

Ploinky Research Agents

copilot-agents is a Ploinky agent repository that integrates Open Interpreter, OpenHands, Agent Laboratory, and The AI Scientist as router-mediated, manifest-discoverable agents. The suite is not a default AssistOSExplorer dependency. It deploys only through an explicit research-agents bundle.

Source of Truth

The DS specifications under docs/specs/ are the source of truth for contracts and invariants. Code, configuration, and tests must follow them. Update specs in the same change as any behavior, interface, or runtime configuration change. Spec-code synchronization is a repository invariant, not an optional documentation task.

Quick Start

cd <workspace-root>
ploinky add repo copilot-agents <repo-url-or-local-path>
ploinky enable repo copilot-agents
ploinky enable agent copilot-agents/research-agents global
ploinky start explorer 8080

Heavy execution agents stay gated behind Ploinky's selectable profiles. The default and dev profiles enable only researchCopilot and openInterpreterAgent; qa adds OpenHands; prod adds OpenHands, Agent Laboratory, and AI Scientist.

Agent Inventory

  • research-agents - explicit deployment bundle
  • researchCopilot - Explorer plugin and dispatch agent
  • openInterpreterAgent - bounded Open Interpreter MCP adapter
  • openHandsAgent - constrained OpenHands adapter (profile-gated)
  • agentLaboratoryAgent - Agent Laboratory phase-driven adapter (profile-gated)
  • aiScientistAgent - AI Scientist adapter, review-first (profile-gated)

AchillesCLI Launchers

Launcher skills live under achilles-skills/. researchCopilot opens AchillesCLI WebChat with dir and skill-root query parameters so that /exec launch-open-interpreter runs a deterministic cskill that returns a valid launch URL. Direct slash aliases such as /open-interpreter require a future generic AchillesCLI dynamic-alias extension and are out of scope for the first slice.
